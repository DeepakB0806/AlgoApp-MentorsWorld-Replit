#164 - Kotak WebSocket Test Harness + Auto Diagnostic Probe
What & Why
When HSM or HSI has a sustained connection failure, there is currently no way to distinguish between:

"Our HSM/HSI code is broken"
"Kotak changed their protocol"
"Kotak's servers are down"
This task builds a permanent, self-contained Kotak demo harness from the official zip (provided by Kotak's engineer) plus an automated diagnostic probe that fires when either gateway hits a sustained failure threshold.

Key principle: The probe uses the raw Kotak demo code (no E41 wire format, no relay, no our abstractions) — a clean-room baseline. If the probe fails, the problem is on Kotak's side. If the probe succeeds but our gateway fails, the problem is in our code.

GET /api/admin/broker-credentials/active — Add to server/routes/admin-routes.ts. Returns {accessToken, sessionId, dataCenter} from the connected broker config. Auth-guarded (superadmin only). This is the endpoint that demo.html's "Load Credentials" button calls.

server/kotak-probe.ts — New module:

interface ProbeResult {
  target: "hsm" | "hsi";
  status: "auth_ok" | "auth_failed" | "unreachable" | "timeout";
  endpoint: string;
  testedAt: string; // ISO timestamp
  durationMs: number;
}

runProbe(config, target): opens a raw ws WebSocket directly to Kotak URL (NO relay — bypassing relay is intentional for a clean-room test)
Sends auth with quotes intact: JSON.stringify({type:"cn", Authorization: config.accessToken, Sid: config.sessionId, source:"WEB"})
Waits up to 10s for {type:"cn", ak:"ok"} → "auth_ok", or error message → "auth_failed", or connection failure → "unreachable", or no response → "timeout"
Stores last result in an in-memory Map<"hsm"|"hsi", ProbeResult> (exported as getLastProbeResults())
Logs [PROBE] verdict clearly
POST /api/admin/probe/run — Add to server/routes/admin-routes.ts. Accepts {target: "hsm"|"hsi"|"both"}. Runs probe immediately, returns result. Used by the dashboard card's "Run Diagnostic Probe" button.

GET /api/admin/probe/status — Returns getLastProbeResults() — last probe result for HSM and HSI. Used by the dashboard card to show last result.

HSI auto-trigger — In server/hsi-kotak-neo-v3.ts, add a consecutiveFailures counter (additive, no locked blocks). Increment after each reconnect attempt that ends without auth_ok. Reset to 0 when {type:"cn", ak:"ok"} received. When counter reaches PROBE_TRIGGER_THRESHOLD (default 5, from env or hardcoded), call runProbe(activeConfig, "hsi") asynchronously (fire-and-forget, never awaited synchronously to avoid blocking reconnect logic).

HSM auto-trigger — Same pattern in server/hsm-kotak-neo-v3.ts. Additive only, no locked blocks.
