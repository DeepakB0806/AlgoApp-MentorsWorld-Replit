Task #95: STRICT FIX - Options LMT Price Resolution & Tick Rounding

What & Why
When placing LIMIT orders for options, the Trade Engine incorrectly uses the index spot price (e.g., NIFTY at 22,500) from the webhook instead of the option's actual premium (e.g., ₹300). This causes immediate RMS rejections. Furthermore, Indian exchanges require options limit prices to be strictly rounded to a 0.05 tick size.

Done looks like

Reverse Token Map: SMC exports a brokerSymbolToTokenMap to quickly find a token by its trading symbol.

Quote API Module: The Execution Layer (el-kotak-neo-v3.ts) has a new getQuote function to fetch the live LTP of a specific token.

Dynamic Base Price: At all 4 LMT call sites in te-kotak-neo-v3.ts, if the order is an Option and the type is LMT, the engine pauses to fetch the real-time LTP via getQuote and uses that as the base price.

Strict Tick Rounding: The final computed limit price (after adding/subtracting the buffer) is strictly rounded to the nearest 0.05 before being sent to the broker.

Task 2: The Execution Layer Quote Fetcher (server/el-kotak-neo-v3.ts)
Export this new function to call Kotak's quotes endpoint:

TypeScript
export async function getQuote(config: BrokerConfig, exchange: string, token: string): Promise<{ success: boolean; ltp?: number; error?: string }> {
  try {
    const endpoint = `/script-details/1.0/quotes/neosymbol/${exchange}|${token}/all`;
    const res = await callKotakApi("GET", endpoint, config); // Use your existing internal API caller
    
    // Extract LTP based on Kotak's standard quote response structure
    const ltp = res?.data?.data?.[0]?.ltp || res?.data?.ltp; 
    
    if (ltp !== undefined && ltp !== null) {
      return { success: true, ltp: Number(ltp) };
    }
    return { success: false, error: "LTP missing in Kotak quote response" };
  } catch (error) {
    return { success: false, error: String(error) };
  }
}

Why this order of operations is critical:
Failsafe Aborts: If the getQuote API call fails (e.g., Kotak's servers glitch), the throw new Error ensures the trade aborts immediately. It will never default back to the dangerous 22,500 spot price.

The Math.round(x * 20) / 20 Formula: This is the universal standard for financial tick size rounding.

308.12 * 20 = 6162.4

Math.round(6162.4) = 6162

6162 / 20 = 308.10 (Perfectly valid tick size).
