Task #97: Error Handling Command Centre (Terminal Routing & UI Failsafes)

What & Why
A blanket, infinite retry loop for all failed orders is a dangerous anti-pattern. The system must differentiate between Transient Errors (like Kotak code 1005: Internal Error or 424: Dependency failure which should be retried) and Terminal Errors (like 1007: Invalid Symbol or expired contracts which are mathematically impossible to retry).
Currently, terminal rejections during exits result in an infinite close_failed zombie retry loop. We need to expose exact Kotak rejection reasons to the UI, implement a strict Terminal Decision Matrix to instantly kill dead loops, and provide a manual UI override.

Done looks like

Accurate UI Data: The Rejection Reason and Time columns in the Order Panel show the actual Kotak rejRsn and ordDtTm fields.

Terminal Routing: When closeTrade() receives a terminal Kotak error code (1006, 1007, 1009) or a known RMS terminal string ("instrument has been expired", "order type is invalid"), it bypasses all retry loops and marks the trade as closed (pnl=0, exitPrice=ltp).

Manual Kill Switch: The UI features a "Force Close" button for close_failed trades, allowing the operator to manually terminate the loop without touching the database.

Task 2 — The Terminal Decision Matrix in closeTrade()
File: server/te-kotak-neo-v3.ts (inside the failure handler for closeTrade after !orderResult.success)

// Exact Kotak RMS strings and API V3 Terminal Codes
const TERMINAL_CLOSE_PATTERNS = [
  "instrument has been expired", // Exact RMS dead contract string
  "order type is invalid",       // Exact RMS invalid contract string
  "1007", "invalid symbol", "scrip not found",
  "1009", "invalid quantity", "wrong lot",
  "1006", "invalid exchange",
  "insufficient holding", "insufficient balance", 
  "no open position", "position not found", "no position",
  "delisted", "suspended", "not available"
];

const errStr = String(orderResult.error || "").toLowerCase();
const isTerminal = TERMINAL_CLOSE_PATTERNS.some(p => errStr.includes(p));

if (isTerminal) {
  const terminalMsg = `[TE] Terminal rejection for ${trade.tradingSymbol}: "${orderResult.error}". Marking as closed to prevent zombie retries.`;
  console.warn(terminalMsg);
  await logPFL(plan.id, trade.entryLegId || "N/A", "force_closed", terminalMsg);
  
  const closedUpdate = await storage.updateStrategyTrade(trade.id, {
    status: "closed", 
    pnl: 0, 
    exitedAt: new Date(), 
    updatedAt: new Date(), 
    exitPrice: trade.ltp || trade.price || 0, 
    exitAction: plan.exitAction || "SELL"
  });
  
  tradingCache.invalidateOpenTrades(trade.planId);
  return closedUpdate || trade;
}
