#239 - Persist TSL Activation State Across Restarts

What & Why
The TSL engine (Task #215) tracks tslActivated in memory. A trade with tslActivateAt = 500 starts with tslActivated = false and only begins trailing once ₹500 profit is reached. But if the server restarts while that trade is live and already activated, the engine rehydrates it from DB with tslActivated = false — it reverts to "waiting" mode and may miss the trailing window entirely.

This is a silent correctness bug: the server logs look normal after restart, but the active trail is no longer advancing its SL, and there's no way to know without checking the DB directly.

Done looks like
tslActivated is stored as a boolean column in strategy_trades
When a trail activates (tslActivated flips from false to true), flushDirtyTrails persists the new value to DB on the next flush cycle (within 15 s)
On server restart, registerNewTrail reads tslActivated from the rehydrated trade record and initializes the trail correctly — already-active trails resume trailing immediately
Trades with tslActivateAt = null / 0 (immediate activation) are unaffected — their tslActivated starts true both in memory and in DB
Out of scope
Changing the flush interval (remains at 15 s)
Persisting highWaterMark mid-trail more frequently (separate concern)
Any UI changes (Task #237 handles display)
Steps
Add tslActivated column to both schema copies — Add tslActivated: boolean("tsl_activated").default(false) to the strategyTrades table in both lib/db/src/schema/schema.ts and artifacts/mentors-world/src/shared/schema.ts. Run pnpm --filter @workspace/db run push to apply.
Persist tslActivated in flushDirtyTrails — In tsl-kotak-neo-v3.ts, extend the updateStrategyTrade call in flushDirtyTrails [TSL-3] to also write tslActivated when the state is dirty. The field only changes once per trade lifetime (false → true), so the extra column write is negligible.
Read tslActivated in registerNewTrail — In registerNewTrail [TSL-2], read (trade as any).tslActivated ?? false when initializing the in-memory TslState. If true, set tslActivated: true and skip the activation gate — the trail resumes trailing immediately using the persisted highWaterMark and currentSlPrice.
