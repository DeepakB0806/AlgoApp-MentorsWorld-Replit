#49 - Batch 4 · TSL Engine

What & Why
The Trailing Stop Loss engine monitors all open NRML trades that have trailingStep > 0. On each 2-second tick, it fetches current LTP via md-kotak-neo-v3.ts, updates highWaterMark and currentSlPrice in the database, and fires startPersistentExit() when live price breaches the active SL level.

OOM Constraint — Zero-History Overwrite (strictly enforced)
This engine reads from the cache and writes to the database. The following rules are absolute:

Fetch the open trades list fresh on every interval tick — do not cache the trade list in a module-level variable between ticks
No price history arrays — do not accumulate LTP readings per instrument. Each tick is processed and discarded.
highWaterMark is computed as Math.max(existingHWM, currentLtp) — a single scalar update, not a running array
Do not write LTP values to any database table — only highWaterMark and currentSlPrice are persisted (they live in strategyTrades columns added in #46)
The only module-level state permitted is the setInterval handle for cleanup purposes
