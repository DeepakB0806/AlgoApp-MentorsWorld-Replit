Task #86: SEBI Limit Orders, UX Risk Fix & Strict "Zero-Hardcode" Architecture
Background & Objective:

SEBI Compliance: SEBI has banned pure Market Orders (MKT) for algos. We must transition to Buffered Limit Orders (LMT).

Strict Architecture: We are enforcing a "Zero Hardcoded Fallbacks" policy. The engine must rely 100% on the database and "Fail-Fast" if a value is missing.

function getBufferedLimitPrice(currentPrice: number, action: string, bufferPoints: number): string {
  if (!currentPrice || currentPrice <= 0) return "0";
  const rawPrice = action.toUpperCase() === "BUY" 
    ? currentPrice + bufferPoints 
    : Math.max(0.05, currentPrice - bufferPoints); 
  const roundedPrice = Math.round(rawPrice * 20) / 20;
  return roundedPrice.toFixed(2);
}
