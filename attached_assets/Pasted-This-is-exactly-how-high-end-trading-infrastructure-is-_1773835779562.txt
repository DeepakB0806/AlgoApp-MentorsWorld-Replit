### The "Hybrid" Data Architecture
Here is how the data flow will work between the two deployments to ensure the seamless failover you described:

1.  **Deployment 1 (Production):** The "Boss." It still has its 10-second API polling loop active.
2.  **Deployment 2 (WebSocket/Fork):** The "Scout." It listens to Kotak’s HSM feed at millisecond speeds.
3.  **The Bridge:** Every time the Scout (WS) gets a new price, it sends a super-fast internal signal to the Boss (Production).
4.  **The Failover:** If the Boss hasn't received a signal from the Scout for more than, say, 2 seconds, it simply uses the price from its own last API poll.
