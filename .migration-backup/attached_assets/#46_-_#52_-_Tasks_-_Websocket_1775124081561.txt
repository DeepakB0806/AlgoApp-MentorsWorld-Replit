#47 - Task #47: Create Settlement Engine (server/se-kotak.ts)
What & Why
deferDailyPnlUpdate in the TE is a setTimeout(1s) that races on concurrent multi-leg closes — each leg reads the same stale base P&L and overwrites each other. The Settlement Engine replaces it with a reliable background sweep using pnlCalculated as an idempotency guard.

Implementation (new file: server/se-kotak.ts)
export function startSettlementEngine(storage: IStorage): void {
  setInterval(async () => {
    // 1. Fetch all closed trades where pnlCalculated = false
    // 2. Group by planId + IST date (from executedAt/exitedAt)
    // 3. For each group: sum pnl, upsert strategyDailyPnl
    // 4. Batch-mark all processed trades pnlCalculated = true
  }, 10_000);
}
Key design points
pnlCalculated flag (from #46) ensures each trade's P&L is counted exactly once
Groups by planId + IST date to correctly bucket multi-leg closes into one daily P&L entry
Upsert: if daily record exists → add delta; if not → create new record
Marks pnlCalculated = true on all processed trades after successful upsert

#49 - Task #49: Create Event-Driven TSL Engine (server/tsl-engine.ts)
What & Why
Kotak only supports TSL for MIS orders. For NRML option-selling positions we need a software TSL that tracks live LTP against each position's trail and fires a market close when the SL is breached.

Architecture
RAM-first: active TSL states in Map<tradingSymbol, TslState[]>
Event-driven: does NOT poll prices internally — receives ticks via processTick()
Dirty flush: setInterval(15s) writes only changed currentSlPrice + highWaterMark to DB
Boot rehydration: loads all open NRML trades with initialSlPrice IS NOT NULL into RAM on start

const WS_STALE_MS = Number(process.env.WS_STALE_MS ?? 2000);
export async function getPrice(symbol: string): Promise<number> {
  const cached = priceCache.get(symbol);
  if (cached && Date.now() - cached.updatedAt < WS_STALE_MS) {
    return cached.ltp; // WebSocket data is fresh — use it
  }
  // Stale or missing — fall back to REST API
  const ltp = await fetchQuoteFromApi(symbol); // calls Kotak getQuote
  priceCache.set(symbol, { ltp, updatedAt: Date.now() });
  return ltp;
}

#53 - Task #53: Kotak HSM WebSocket Gateway (server/ws-kotak.ts)
What & Why
Connects to Kotak Neo's HSM (HyperSync Market) WebSocket feed. Pushes every incoming price tick into the Market Data Manager RAM cache so getPrice() returns sub-second data. If the WebSocket drops, the Market Data Manager automatically falls back to REST — no intervention needed.

Architecture
Singleton WebSocket client, module-level
Connects to Kotak HSM using sid, token, dataCenter from broker config
On each tick: calls marketData.updatePrice(symbol, ltp)
Auto-reconnect: silent exponential backoff on disconnect (1s → 2s → 4s → max 30s)
subscribe(symbol) — sends HSM subscription message for a given scrip token
startWsGateway(storage) — called at boot, connects WS + auto-rehydrates subscriptions
Kotak HSM connection
// Protocol: HSM WebSocket (Kotak Neo V3 spec)
// Endpoint: wss://mlhsm.kotaksecurities.com  (or equivalent from broker config)
// Auth: sid + access_token from active broker session
// Message format: { type: "subscribe", tokens: ["26000", "35001", ...] }
Auto-reconnect logic
function connect() {
  ws = new WebSocket(HSM_URL);
  ws.on("open", () => {
    log("WS Gateway: connected to Kotak HSM");
    resubscribeAll(); // re-send all active subscriptions after reconnect
  });
  ws.on("message", (data) => {
    const tick = parseTick(data);
    if (tick) marketData.updatePrice(tick.symbol, tick.ltp);
  });
  ws.on("close", () => {
    log("WS Gateway: disconnected — reconnecting in ${backoffMs}ms");
    setTimeout(connect, backoffMs);
    backoffMs = Math.min(backoffMs * 2, 30000);
  });
  ws.on("error", (err) => {
    log("WS Gateway error:", err.message); // silent — close event will trigger reconnect
  });
}
subscribe(symbol) — maps symbol to scrip token
export function subscribe(symbol: string): void {
  const token = scriptTokenMap.get(symbol); // resolved from SMC (Scrip Master Cache)
  if (!token) { log(`WS: no token for ${symbol}`); return; }
  activeSubscriptions.add(symbol);
  if (ws?.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify({ type: "subscribe", tokens: [token] }));
  }
  // If WS not yet open, resubscribeAll() on connect will catch it
}
startWsGateway(storage) — boot entry point
export async function startWsGateway(storage: IStorage): Promise<void> {
  // 1. Load open NRML trades and subscribe their symbols
  const openTrades = await storage.getOpenNrmlTradesWithTsl();
  const symbols = [...new Set(openTrades.map(t => t.tradingSymbol).filter(Boolean))];
  for (const symbol of symbols) subscribe(symbol);
  log(`WS Gateway: auto-subscribed ${symbols.length} symbols from open trades`);
  // 2. Connect WebSocket
  connect();
}
parseTick() — internal
Parses the raw Kotak HSM message format (JSON or binary depending on HSM version) into { symbol: string, ltp: number }. Uses the existing SMC token-to-symbol reverse map.

Non-fatal on failure
If startWsGateway cannot connect (bad credentials, network), it logs the error and returns without throwing. The rest of the app continues using REST API polling via Market Data Manager.
