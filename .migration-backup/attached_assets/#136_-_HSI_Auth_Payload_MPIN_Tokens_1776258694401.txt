#136 - HSI Auth Payload: MPIN Tokens & Strict JSON Schema

What & Why
The Kotak Interactive stream (HSI) on the E41 datacenter is actively rejecting the WebSocket authentication payload. Deep analysis of the official Kotak Python SDK (Websocket_KotakNeo.ipynb) reveals two critical payload requirements that our raw implementation currently violates:

The MPIN Token Inference: The Colab notebook proves Kotak's login is a strict two-step process. Step 1 (totp_login) issues a token with "scope":["View"]. Step 2 (totp_validate with MPIN) issues the final session token with "scope":["Trade"] and explicitly assigns the E41 datacenter. Because HSI is the order-execution stream on E41, it strictly requires the "Trade" token (accessToken / sessionId), not the "View" tokens.

The "src" Schema Bug: The Kotak WebSocket parser strictly expects the payload key "src": "WEB". Our current buildAuthMessage sends "source": "WEB". The E41 server cannot parse this, immediately dropping the connection with the error: {"ak":"nk","type":"failed to connect","msg":"session message format incorrect"}.
