#169 - Full-Stack Trailing Stop Loss (TSL) — Schema + Engine + UI
Price Architecture — mandatory for all engines in this task
Every price lookup in TSL, MTM Monitor, and any future engine MUST go through getPrice() in server/md-kotak-neo-v3.ts. Never read priceCache directly.

getPrice() is a LOCKED 3-tier fallback [MD-1]:

WS cache (< 2s fresh) — fed by HSM heartbeat via updatePrice(symbol, ltp). This is the primary path. When HSM is healthy, every tick lands here within milliseconds.
REST fallback — if cache is stale (HSM slow/disconnected), calls EL.getQuote(config, exchange, token) directly against the broker REST API. Requires brokerConfig, exchange, and token to be passed into getPrice().
Stale cache — if REST also fails, returns the last known price rather than null. Last resort, never discarded.
HSI note: HSI receives position and trade events (which carry LTP) but currently only logs them — it does not call updatePrice(). This means HSI is not yet a price source for the cache. A future task could wire HSI position LTPs into priceCache, but that is out of scope here. HSM is the sole WS price source for now.

Task #138 dependency: Task #138 fixes the HSM subscription mechanism ({type:"mws"} correction). Until #138 is merged, HSM subscriptions may silently fail for some symbols, causing the REST fallback to activate frequently. Task #169 must be built to tolerate this — by always calling getPrice() with full broker/token args so REST fires when needed.

Rule for all engines in this task:

TSL engine (processTick path): call getPrice(symbol, brokerConfig, exchange, token) — not priceCache.get()
MTM monitor: call getPrice(symbol, brokerConfig, exchange, token) for each open trade's symbol
Both must pass full args so REST fallback can activate if HSM cache is stale
Both must handle null return (all 3 tiers failed) by skipping that symbol gracefully — never crash, never assume a price

TSL tick frequency — critical fix bundled here
The plan-monitor currently drives TSL ticks every 60 seconds — far too slow for a trailing stop loss that needs to track price movement continuously. The MTM monitor (5s interval) will take over TSL tick driving, making the plan-monitor's TSL poll redundant.

In server/plan-monitor.ts, remove the TSL polling block (lines ~150-175 that call getOpenNrmlTradesWithTsl() and processTick()). The MTM monitor replaces it with a 5s driven path.

The MTM monitor will call processTick(symbol, ltp) for every open TSL-enabled trade on every 5s cycle, using getPrice() (MD-1, 3-tier fallback). This means:

HSM healthy → WS cache hit → processTick fires within 5s of every price move
HSM stale/down → REST EL.getQuote fires per symbol → processTick still fires
Both tiers failed → skip that symbol that cycle, log warning — never crash

// Interval: 5000ms
// For each active/deployed plan that has stoploss.enabled OR profitTarget.enabled:
//   1. Load open strategy_trades for the plan
//   2. For each open trade, get live LTP from priceCache
//   3. Compute per-leg live MTM and sum into planMTM
//   4. Resolve thresholds (amount mode: direct INR; percentage mode: % of capital)
//   5. If planMTM <= -stoplossThreshold → fire square-off
//   6. If planMTM >= profitTargetThreshold → fire square-off

Detailed logic:

// Price access rule: ALWAYS call getPrice() with full args — never priceCache directly.
// getPrice() is MD-1 (locked): WS cache → REST EL.getQuote → stale cache.
// Requires brokerConfig, exchange, token per symbol for REST fallback to activate.
// token = brokerSymbolToTokenMap.get(trade.tradingSymbol) (same pattern as plan-monitor TSL poll)
async function computePlanMTM(openTrades, brokerConfig):
  capital = 0
  mtm = 0
  for each trade in openTrades:
    token = brokerSymbolToTokenMap.get(trade.tradingSymbol)
    ltp = await getPrice(trade.tradingSymbol, brokerConfig, EL.mapExchange(trade.exchange), token)
    if ltp === null: skip this leg — all 3 tiers failed, do not assume a price
    entryPrice = trade.price ?? 0
    qty = trade.quantity ?? 0
    legMTM = trade.action === "BUY"
               ? (ltp - entryPrice) * qty
               : (entryPrice - ltp) * qty
    mtm += legMTM
    capital += entryPrice * qty
  return { mtm, capital }
function resolveThreshold(configValue, mode, capital):
  if mode === "amount": return configValue
  if mode === "percentage": return (capital * configValue) / 100
async function checkAll(storage):
  // ── PART A: Drive TSL ticks (replaces plan-monitor's 60s TSL poll) ──
  tslTrades = await storage.getOpenTradesWithTsl()   // renamed in Step 3
  brokerConfigCache = new Map()
  for each trade of tslTrades:
    plan = await storage.getStrategyPlan(trade.planId)
    if !plan?.brokerConfigId: continue
    bc = brokerConfigCache.get(plan.id) ?? await storage.getBrokerConfig(plan.brokerConfigId)
    brokerConfigCache.set(plan.id, bc)
    token = brokerSymbolToTokenMap.get(trade.tradingSymbol)
    ltp = await getPrice(trade.tradingSymbol, bc, EL.mapExchange(trade.exchange), token)
    if ltp === null:
      log(`[MTM-MONITOR] price unavailable for ${trade.tradingSymbol} — all 3 tiers failed, skipping tick`)
      continue
    processTick(trade.tradingSymbol, ltp)   // drives TSL engine
  // ── PART B: Plan-level MTM stoploss + profit target check ──
  plans = await storage.getStrategyPlans()
  activePlans = plans.filter(p =>
    (p.deploymentStatus === "active" || p.deploymentStatus === "deployed") && p.brokerConfigId
  )
  for each plan of activePlans:
    tradeParams = parseTradeParams(plan)
    slCfg = tradeParams?.stoploss
    ptCfg = tradeParams?.profitTarget
    if not (slCfg?.enabled || ptCfg?.enabled): continue
    if persistentSquareOffActive.has(plan.id): continue  // already squaring off
    openTrades = await storage.getUnclosedTradesByPlan(plan.id)
    if openTrades.length === 0: continue
    bc = brokerConfigCache.get(plan.id) ?? await storage.getBrokerConfig(plan.brokerConfigId)
    brokerConfigCache.set(plan.id, bc)
    { mtm, capital } = await computePlanMTM(openTrades, bc)  // uses getPrice() internally
    shouldSquareOff = false
    reason = ""
    if slCfg?.enabled && slCfg.value > 0:
      threshold = resolveThreshold(slCfg.value, slCfg.mode, capital)
      if mtm <= -threshold:
        shouldSquareOff = true
        reason = `MTM Stoploss hit: MTM=${mtm.toFixed(0)}, threshold=-${threshold.toFixed(0)}`
    if ptCfg?.enabled && ptCfg.value > 0 && !shouldSquareOff:
      threshold = resolveThreshold(ptCfg.value, ptCfg.mode, capital)
      if mtm >= threshold:
        shouldSquareOff = true
        reason = `MTM Profit Target hit: MTM=${mtm.toFixed(0)}, threshold=${threshold.toFixed(0)}`
    if shouldSquareOff:
      log(`[MTM-MONITOR] ${reason} — plan ${plan.id}`)
      writeProcessFlowLog(storage, plan, reason, "mtm_monitor", "square_off")
      startPersistentSquareOff(storage, plan.id, bc)
